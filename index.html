Projects

PhotoSense: AI Face Censoring

For my senior design project, I worked with a team of 5 to develop a full-stack AI face censoring system. We trained a Keras model based on the UNET convolutional neural network (CNN) architecture to segment faces in images, deploying it to an AWS EC2 instance as a RESTful API for backend image processing. We then built a web application using React, using our Flask API to segment user images upon upload — providing tools to select censoring algorithms, remove metadata tags, and edit segmentations using custom web-based drawing tools that we built. I also enabled our AI censoring service on social media, developing a Twitter bot with Python & Tweepy to allows users to retrieve censored versions of images found in tweets by simply tagging our bot as a reply.

US Chemical Release Visualizer

I submitted a project proposal to the CSULB ACM chapter to develop a geospatial data visualization tool for the EPA Toxic Release Inventory (TRI), landing me a project management position. After selecting 2 team members, we began developing a web mapping application using React & Mapbox JS. I helped develop the application's backend service, storing the EPA data in a PostgreSQL database, leveraging PostGIS to enable geospatial queries, and using Express to build a RESTful API to fetch data on the client. I designed the React component hierarchy to ensure optimal data flow, used React-Vis to render interactive chart components, and created Mapbox JS layers to display icons and tooltip popups for waste facilities found in US cities.

Spot A Bubble

I developed a React.js web application using D3.js and the Spotify API to visualize listening data in an interactive bubble chart based on top genres and artists found in user accounts. Each bubble represents a genre whose radius is determined by how many top artists in a user account correspond to the genre. The chart uses a D3.js force simulation object for the chart's bubble physics as well as its collision effects. Tooltip displays are provided to view the artist data for a given bubble. The app also uses oAuth2.0 for user authentication and Heroku as a deployment service.
